redis概览

从整体上概览Redis的功能。

数据结构与应用

Redis中的5种基本数据类型：
字符串、散列、列表、集合、有序集合
4种特殊数据结构:
位图、地理坐标、HyperLogLog、流

应用

  1. 实现分布式锁：
    • 获取锁：set lock “locked” NX
    • 释放锁：DEL lock
    • 判断锁是否已经获取：判断SET操作是否成功
    • 判断锁是否已经释放：判断DEL操作的返回值是否为1
      （DEL:接受一个或多个数据库键作为参数，尝试删除这些键以及与之相关联的值，并返回被删除键的数量作为结果）

数据库命令

命令 描述 示例
SELECT 切换到指定数据库
KEYS 返回与匹配符相匹配的键
SCAN 以渐进方式迭代数据库中的键
RANDOMKEY 随机返回一个键
SORT 对键的值进行排序
EXISTS 检查给定键是否存在
DBSIZE 获取数据中包含键值对的数量
TYPE 查看键的类型
RENAME 修改键名
RENAMENX 只有当新键名未被占用的情况下修改键名
MOVE 将给定的键移动到另一个数据库
DEL 移除指定的键，同步方式
UNLINK 移除指定的键，异步方式
FLUSHDB 清空当前数据库
FLUSHALL 清空所有数据库
SWAPDB 互换数据库

自动过期

场景

  • 网页缓存
  • 用户会话信息
  • 聚合计算时大量的临时数据

相关命令

  • 设置生存时间的命令

    命令 描述 示例
    EXPIRE 设置秒级精度的生存时间
    PEXPIRE 设置毫秒级别的生存时间

    或者使用SET命令的EX和PX选项

    这两个选项更重要的是保证了操作的原子性，使得“为键设置值”和“为键设置生存时间”这两个操作可以一起执行。

  • 设置过期时间的命令

    命令 描述 示例
    EXPIREAT 当系统的当前UNIX时间超过命令指定的UNIX时间时，给定的键就会被移除，秒级精度
    PEXPIREAT 毫秒级精度

  • 获取键的生存时间的命令

    命令 描述 示例
    TTL 以秒为单位返回键的生存时间
    PTTL 以毫秒为单位返回键的生存时间
    如果给定的键存在，但是没有设置生存时间或者过期时间，这两个命令将返回-1
    如果给定的键不存在，这两个命令将返回-2
